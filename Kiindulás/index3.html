eMenza - Backend és Adatbázis Tervezés

Rendszerarchitektúra

Az eMenza rendszer háromrétegű architektúrát követ, amely a következő komponensekből áll:

Frontend (Vue.js)

Reszponzív felhasználói felület, amely kommunikál a backend API-val RESTful webszolgáltatásokon keresztül.

  • Felhasználói hitelesítés és autorizáció
  • Menü megjelenítése és rendelések kezelése
  • Adminisztrációs felület

Backend (Node.js + Express)

A szerveroldali logika kezeli az üzleti szabályokat, hitelesítést és adatbázis-műveleteket.

  • RESTful API végpontok
  • JWT token alapú hitelesítés
  • Adatvalidáció és biztonság
  • Üzleti logika implementációja

Adatbázis (MySQL)

Relációs adatbázis, amely tárolja a felhasználói adatokat, ételeket, rendeléseket és egyéb információkat.

  • Felhasználók és jogosultságok kezelése
  • Menük és ételek tárolása
  • Rendelések és tranzakciók nyomon követése
  • Értékelések és statisztikák

Külső Integrációk

Harmadik féltől származó szolgáltatások integrációja a rendszerrel.

  • RFID olvasó API integráció
  • Fizetési rendszer (bankkártya, online fizetés)
  • Email értesítések (SendGrid, Mailgun)
  • Mobil alkalmazás REST API

API Végpontok Tervezete

Az alábbiakban az eMenza rendszer főbb REST API végpontjai láthatók:

Hitelesítés

POST /api/auth/register - Felhasználó regisztráció
POST /api/auth/login - Bejelentkezés és JWT token kiadása
GET /api/auth/profile - Felhasználói profil lekérése

Menü kezelés

GET /api/menu - Heti menü lekérése
GET /api/menu/:date - Adott napi menü lekérése
POST /api/menu - Új menü hozzáadása (admin)

Rendelések

GET /api/orders - Felhasználó rendeléseinek lekérése
POST /api/orders - Új rendelés leadása
PUT /api/orders/:id - Rendelés módosítása
DELETE /api/orders/:id - Rendelés törlése (lemondás)

Admin felület

GET /api/admin/users - Összes felhasználó lekérése
GET /api/admin/statistics - Rendszerstatisztikák
POST /api/admin/meals - Új étel hozzáadása

Adatbázis Tervezés

Az eMenza rendszer az alábbi MySQL táblákat fogja használni:

Felhasználók tábla (users)

Mező Típus Leírás
id INT (PK, Auto Increment) Felhasználó egyedi azonosítója
email VARCHAR(255) Felhasználó email címe
password VARCHAR(255) Titkosított jelszó
name VARCHAR(100) Felhasználó teljes neve
role ENUM('user', 'admin') Felhasználó szerepköre
balance DECIMAL(10,2) Felhasználó egyenlege
created_at TIMESTAMP Felhasználó létrehozásának ideje

Ételek tábla (meals)

Mező Típus Leírás
id INT (PK, Auto Increment) Étel egyedi azonosítója
name VARCHAR(255) Étel neve
description TEXT Étel leírása
price DECIMAL(10,2) Étel ára
allergens JSON Allergének listája
diet_info JSON Diétás információk

Rendelések tábla (orders)

Mező Típus Leírás
id INT (PK, Auto Increment) Rendelés egyedi azonosítója
user_id INT (FK) Felhasználó azonosítója
meal_id INT (FK) Étel azonosítója
order_date DATE Rendelés dátuma
status ENUM('active', 'cancelled', 'completed') Rendelés állapota
created_at TIMESTAMP Rendelés létrehozásának ideje

Értékelések tábla (ratings)

Mező Típus Leírás
id INT (PK, Auto Increment) Értékelés egyedi azonosítója
user_id INT (FK) Felhasználó azonosítója
meal_id INT (FK) Étel azonosítója
rating INT(1) Értékelés (1-5)
comment TEXT Szöveges értékelés
created_at TIMESTAMP Értékelés létrehozásának ideje

Kódpéldák

Az alábbiakban néhány fontos kódrészlet látható a backend implementációból:

Express.js Szerver Alapkonfiguráció

const express = require('express'); const cors = require('cors'); const mysql = require('mysql2'); const bcrypt = require('bcrypt'); const jwt = require('jsonwebtoken'); const app = express(); const PORT = process.env.PORT || 3000; // Middleware app.use(cors()); app.use(express.json()); // Adatbázis kapcsolat const db = mysql.createConnection({ host: process.env.DB_HOST || 'localhost', user: process.env.DB_USER || 'root', password: process.env.DB_PASSWORD || '', database: process.env.DB_NAME || 'emenza' }); // Kapcsolat ellenőrzése db.connect((err) => { if (err) { console.error('Database connection failed: ' + err.stack); return; } console.log('Connected to database as id ' + db.threadId); }); // Alap útvonal app.get('/', (req, res) => { res.json({ message: 'eMenza API Running' }); }); // Szerver indítása app.listen(PORT, () => { console.log(`Server is running on port ${PORT}`); });

Hitelesítési Részleg

// Regisztráció app.post('/api/auth/register', async (req, res) => { try { const { email, password, name } = req.body; // Ellenőrzés, hogy létezik-e már a felhasználó const userExists = await db.execute( 'SELECT id FROM users WHERE email = ?', [email] ); if (userExists[0].length > 0) { return res.status(400).json({ error: 'User already exists' }); } // Jelszó hash-elés const saltRounds = 10; const hashedPassword = await bcrypt.hash(password, saltRounds); // Felhasználó létrehozása const result = await db.execute( 'INSERT INTO users (email, password, name, role, balance) VALUES (?, ?, ?, "user", 0)', [email, hashedPassword, name] ); res.status(201).json({ message: 'User created successfully' }); } catch (error) { res.status(500).json({ error: 'Internal server error' }); } }); // Bejelentkezés app.post('/api/auth/login', async (req, res) => { try { const { email, password } = req.body; // Felhasználó keresése const [users] = await db.execute( 'SELECT * FROM users WHERE email = ?', [email] ); if (users.length === 0) { return res.status(400).json({ error: 'Invalid credentials' }); } const user = users[0]; // Jelszó ellenőrzése const validPassword = await bcrypt.compare(password, user.password); if (!validPassword) { return res.status(400).json({ error: 'Invalid credentials' }); } // JWT token generálása const token = jwt.sign( { userId: user.id, email: user.email, role: user.role }, process.env.JWT_SECRET || 'your_secret_key', { expiresIn: '24h' } ); res.json({ token, user: { id: user.id, email: user.email, name: user.name, role: user.role } }); } catch (error) { res.status(500).json({ error: 'Internal server error' }); } });

Middleware JWT Ellenőrzéshez

// Middleware a token ellenőrzéséhez const authenticateToken = (req, res, next) => { const authHeader = req.headers['authorization']; const token = authHeader && authHeader.split(' ')[1]; if (!token) { return res.status(401).json({ error: 'Access token required' }); } jwt.verify(token, process.env.JWT_SECRET || 'your_secret_key', (err, user) => { if (err) { return res.status(403).json({ error: 'Invalid or expired token' }); } req.user = user; next(); }); }; // Példa védett útvonal app.get('/api/profile', authenticateToken, async (req, res) => { try { const [users] = await db.execute( 'SELECT id, email, name, role, balance FROM users WHERE id = ?', [req.user.userId] ); if (users.length === 0) { return res.status(404).json({ error: 'User not found' }); } res.json(users[0]); } catch (error) { res.status(500).json({ error: 'Internal server error' }); } });
